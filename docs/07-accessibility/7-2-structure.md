# 7-2. 構造と見出し <Badge type="danger" text="必須" />

## 7-2-1. 見出し階層の適切な使用（達成基準 1.3.1）

`h1`〜`h6` 要素の階層を適切に使用し、視覚的なデザインだけで見出しを表現しないようにします。ページに `h1` は原則1つとします。

## 7-2-2. リストの使用（達成基準 1.3.1）

ナビゲーションメニューや複数の項目の列挙には、`ul` / `ol` / `li` 要素を使用します。

## 7-2-3. テーブルの構造（達成基準 1.3.1）

テーブルにはヘッダーセルに `th` 要素と `scope` 属性を使用し、`caption` 要素でテーブルの説明を付与します。

## 7-2-4. 見出しレベルのスキップ禁止

見出しレベルを飛ばして使用してはなりません。スクリーンリーダーのユーザーは見出しを手がかりにページを探索するため、レベルの欠落は文書構造の破綻として認識されます。

デザイン上のフォントサイズ調整のために見出しレベルを変えることを禁止します。見た目の調整はCSSで行います。

::: code-group

```html [NG]
<!-- h2の次にh4が来ている（h3がスキップされている） -->
<h2>サービス一覧</h2>
<h4>Webデザイン</h4>
```

```html [OK]
<!-- 順番にレベルを使う -->
<h2>サービス一覧</h2>
<h3>Webデザイン</h3>
```

:::

## 7-2-5. ランドマークと aria-label

`header` / `nav` / `main` / `footer` などのランドマーク要素を適切に使用します。これにより、スクリーンリーダーのユーザーがページの主要な領域に素早くジャンプできるようになります。

同一ページ内に同じランドマーク要素が複数存在する場合は、`aria-label` または `aria-labelledby` でそれぞれを区別します。

::: code-group

```html [NG]
<!-- navが複数あるが、どちらか区別できない -->
<nav>...</nav>
<nav>...</nav>
```

```html [OK]
<!-- aria-labelで役割を明示する -->
<nav aria-label="グローバルナビゲーション">...</nav>
<nav aria-label="ページ内目次">...</nav>
```

:::

見出し要素を使って区別する場合は `aria-labelledby` を使います。

```html
<nav aria-labelledby="sitemap-heading">
  <h2 id="sitemap-heading">サイトマップ</h2>
  ...
</nav>
```
