# 4-2. セマンティクスと構造 <Badge type="danger" text="必須" />

## 4-2-1. 構成要素の分離

文書構造・表現・挙動はそれぞれ HTML・CSS・JavaScript に分離します。表現に関するものはスタイルシートに、挙動に関するものはスクリプトに記述します。CSSアニメーション・ホバーエフェクトなどはCSSで表現します。

## 4-2-2. type属性の省略

CSSとJavaScriptのtype属性はHTMLではデフォルトとして解釈されるため、省略します。

::: code-group

```html [NG]
<link rel="stylesheet" href="style.css" type="text/css">
<script src="app.js" type="text/javascript"></script>
```

```html [OK]
<link rel="stylesheet" href="style.css">
<script src="app.js"></script>
```

:::

## 4-2-3. メインコンテンツの明示

ヘッダー・フッターなど共通部分を除いたページのメインコンテンツは `main` 要素でマークアップします。

## 4-2-4. 装飾目的のタグ利用禁止

必要以上に `div` で囲む、背景画像のために空の `div` を利用する、レイアウト目的で `table` を使用するなどの行為を避けます。視覚表現は可能な限りCSSに分離し、HTMLでは構造表現のみを記述します。

## 4-2-5. 区切り記号・スペースの扱い

リンクの区切り記号「｜」やパンくずリストの階層表示記号「>」などテキストとして意味を持たない記号は、背景画像やCSSの疑似要素などで表現します。また、要素間の空き調整に `&nbsp;` や全角スペースを使用してはなりません。

## 4-2-6. loading="lazy" の活用

ファーストビューより下に位置する `img` 要素には、原則 `loading="lazy"` を指定します。表示領域に入るまで画像の読み込みを遅延させることで、初期ページロードの通信量を削減できます。

- **ファーストビュー内の画像**（ヒーロー画像など）には指定しません。遅延が発生して表示が遅くなります。
- `iframe` にも同様に適用できます。

```html
<!-- ファーストビュー内: lazy不要 -->
<img src="/img/hero.jpg" alt="メインビジュアル">

<!-- ファーストビュー以下: lazyを付与 -->
<img src="/img/news-thumb.jpg" alt="お知らせサムネイル" loading="lazy">
```

## 4-2-7. br要素の利用制限

`br` 要素は、改行そのものが意味を持つ場合（詩・俳句・住所など）にのみ使用します。デザイン上の折り返し位置を調整する目的で使用してはなりません。

レスポンシブ対応や多言語化を考慮すると、`br` による改行はCMSの入力データや画面幅の変化に対して脆弱です。視覚的な折り返し制御はCSSで行います。

::: code-group

```html [NG]
<!-- デザイン調整のためにbrを使っている -->
<p>お問い合わせは<br class="sp-only">こちらから</p>
```

```html [OK]
<!-- spanとCSSで制御する -->
<p>お問い合わせは<span class="u-block-sp">こちらから</span></p>
```

:::

```css
/* SP時のみ改行 */
@media (max-width: 767px) {
  .u-block-sp {
    display: block;
  }
}
```
