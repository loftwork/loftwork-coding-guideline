# 9-1. メンテナンスの継続性 <Badge type="danger" text="必須" />

JavaScript・CSSを問わず、外部ライブラリを導入する際は以下の基準を満たしていることを確認します。

セキュリティ上のリスクや、ブラウザ・OSのアップデートへの追従を考慮し、開発・保守が継続されているライブラリを選択します。

**確認の目安：**

- GitHubなどのリポジトリで、直近1年以内にコミットまたはリリースがある
- IssueやバグレポートへのReplyが継続されている
- README・CHANGELOGが更新されている

> 長期間メンテナンスが止まっているライブラリは、脆弱性への対処がおこなわれない可能性があります。導入前に `npm audit` やGitHubのSecurity Advisoriesも確認してください。
